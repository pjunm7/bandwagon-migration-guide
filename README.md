## 搬瓦工机房迁移是什么

搬瓦工的机房迁移（Migrate to another DC）是 KiwiVM 控制面板里的一个功能，允许你把现有 VPS 从一个数据中心搬到另一个，整个过程不需要重新购买、不会丢数据，IP 地址会跟着变，其余配置照旧。

迁移完成后，原来跑的服务换个 IP 就可以继续用，SSH 密码、系统、文件都还在。

这个功能对国内用户来说特别实用——线路表现因地区、运营商而异，买回来先在几个机房跑一圈，找到延迟最低的那个再固定下来，比买前纯靠猜靠测试 IP 准很多。

## 为什么有的套餐不支持迁移

讲真，这背后是商业逻辑。

搬瓦工的限量版套餐普遍比常规套餐便宜很多——同等配置，限量版有时候是常规套餐价格的三分之一甚至更低。价格打下去了，代价是机房锁死。你选了 DC6 就只能在 DC6 用，不能迁到 DC9、不能迁到日本软银、不能迁到香港。

这样搬瓦工可以针对特定机房的资源做集中规划，卖出去的量也更可控。用户端拿到了低价，失去了灵活性。

常规套餐价格高，但在可选机房范围内随便切，迁多少次没有次数限制（迁移本身会消耗一点流量）。

搞清楚这个逻辑，后面选套餐的时候就能对应自己的需求了。

## 怎么判断自己买的套餐支不支持迁移

两种方法：

**方法一（买之前）**：在套餐详情页或搬瓦工官网定价页，搜索关键词"migrate"或看套餐说明里是否提到"Migrate to another DC"。限量版套餐的说明通常会写"暂不支持迁移机房"。

**方法二（买之后）**：登录 KiwiVM 控制面板，看左侧菜单有没有"Migrate to another DC"这一项。有就可以迁，没有就是当前套餐不支持。

## 搬瓦工机房迁移操作步骤（支持迁移的套餐）

1. 登录搬瓦工官网，点击顶部菜单 **Services → My Services**
2. 找到对应的 VPS 实例，点击 **KiwiVM Control Panel**
3. 进入 KiwiVM 面板后，左侧找到 **Migrate to another DC**
4. 右侧会列出当前可迁移的机房列表，选择目标机房
5. 点击 **Confirm on next step**，然后点击 **Start Migration** 开始迁移
6. 等待几分钟，迁移完成后 IP 会变更，数据和配置不变
7. 更新域名解析或本地 SSH 配置里的 IP，重新连接

注意：迁移前最好先创建一个快照，以防万一。迁移本身会消耗少量流量，不影响主要的月度流量配额，但有具体的消耗计算规则，大概是 VPS 容量的若干倍。

## FAQ

**Q：搬瓦工的香港 CN2 GIA 套餐能不能迁移到其他机房？**

不能。无论是月付 $89.99 起的常规香港 CN2 GIA（HKHK_8），还是限量版香港 HK85（HKHK_3），都不支持迁出去。买香港就是认准香港这一个机房用的。

**Q：KVM 套餐可以迁移到 CN2 GIA 机房吗？**

不行。KVM 套餐和 CN2 GIA-E 套餐是两个不同级别的系列，低级别不能迁到高级别机房。想用 CN2 GIA 机房，要买 CN2 GIA-E 系列或更高的套餐。

**Q：迁移机房次数有限制吗？**

次数本身没有限制，但每次迁移会消耗一定流量。不影响正常使用，但不建议频繁迁移。

**Q：不支持迁移的套餐，IP 被墙了怎么办？**

IP 被墙后，同样可以申请换 IP（付费 $8，或者符合条件的情况下免费），但机房本身不能换。如果当前机房延迟已经没问题只是 IP 被封，换个 IP 就能继续用。

**Q：THE PLAN 2024 限量版现在有货吗？**

该套餐属于限量补货型，不是常规在售，补货后几小时内通常售罄。可以直接去搬瓦工官网查看当前库存状态。
